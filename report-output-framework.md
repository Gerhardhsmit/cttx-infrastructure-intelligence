# Reserve Manager Report Output Framework

The upgraded client-facing report should answer four practical questions for a reserve manager: what CTTX found, what the findings mean, what should happen next, and who is responsible for the next action. The report must remain clear that early scores are **preliminary intelligence**, not a final RF survey or contractual network design.

| Report Section | Purpose | Client-Facing Answer |
| --- | --- | --- |
| Executive Summary | Convert scores into plain language. | “Your intake shows whether a practical connectivity path is likely and which risks need field validation.” |
| Recommended Next Actions | Translate scoring and pins into steps. | “Validate the handoff point, survey the priority zones, confirm resilience requirements, and price the preferred architecture.” |
| Who Contacts Whom | Remove ambiguity after submission. | “CTTX reviews the intake and should contact the listed email; the reserve manager should prepare maps, gate access, current provider details, and power information.” |
| Engineering Follow-up | Explain the next operational stage. | “The next step is a field/desktop engineering review that verifies line-of-sight, backhaul availability, distribution zones, power, and commercial viability.” |
| Decision Checklist | Help the manager act internally. | “Approve a discovery call, nominate a site contact, share current connectivity bills/provider data, and confirm high-priority operational zones.” |

The recommendations should be deterministic and visible on every generated report, even if the audit data is incomplete. When data is incomplete, the copy should shift from confident recommendation to “information needed next” so the manager still knows what to do.
